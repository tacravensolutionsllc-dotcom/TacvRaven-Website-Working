FAQ

Frequently Asked Questions

Common questions about our threat intelligence tools and how they work.

BlackFeather delivers intelligence community-grade assessments by combining log analysis with MITRE ATT&CK mapping, structured analytical techniques, and APT pattern recognition. It cross-references your data against known threat actor behaviors to identify patterns and produce rigorous, defensible assessments — all in a single file that works completely offline.

BlackFeather ingests log data (via Corvus Engine or direct input), maps activities to the MITRE ATT&CK framework, applies structured analytical techniques used by the intelligence community, and cross-references against APT campaign data. This produces assessments with ICD 203 compliant confidence levels — the same standard used by U.S. intelligence agencies.

Structured analytical techniques (SATs) are methodologies developed by the intelligence community to reduce bias and improve analytical rigor. They include techniques like Analysis of Competing Hypotheses (ACH), key assumptions checks, and indicators analysis. BlackFeather integrates these techniques to help analysts move beyond gut instinct to evidence-based conclusions.

Single-file deployment eliminates IT friction. No installation approvals, no dependency management, no network configuration. Copy one HTML file to any system — air-gapped, classified, or standard — and it just works. This is critical for incident responders who need tools immediately, or organizations with strict software approval processes.

Corvus Engine handles Windows EVTX binary files, Syslog, and various custom log formats. It normalizes messy, inconsistent log data into clean, structured output optimized for BlackFeather analysis. If you're dealing with logs that don't play nice together, Corvus Engine is your preprocessing solution.
